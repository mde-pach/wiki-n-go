# Talk pages

Alongside every article sits a **talk page** — a discussion area for the conversation *around* a page rather than the page itself. It is where you ask a question, propose a change you're unsure about, or work out wording with someone before touching the article. Wikipedia editors have leaned on the same idea for years; it's what keeps disagreement off the page and in a place built for it.

You reach it through the **Discussion** (or **Talk**) tab that every page carries. Open a topic with a title and a first message, and others reply beneath it. Conversations stay in threads, each topic shows how many replies it has and when it last stirred, and every message has its own link you can point someone at.

## What's underneath

Talk pages are built on [[w:GitHub|GitHub]] Discussions. When you post, the comment is stored as a Discussion in the wiki's repository and shown back to you in the page's own styling — so the talk layer needs no database of its own, and the whole conversation history travels with the repo like everything else. You can post anonymously or signed in, matching how you'd [[editing-model|edit]].

## When to talk instead of edit

A wiki runs on edits, not discussion, so the honest default is: if you can just improve the page, do that. Talk pages are for the cases where editing first would be rude or risky.

- You want a second opinion before making a change you're unsure about.
- You've spotted a problem but don't want to fix it yourself.
- Two people disagree about wording and would rather settle it than edit over each other.

Used that way, the talk page keeps the article calm and the argument productive. Used as a substitute for editing, it just becomes a forum nobody reads.
