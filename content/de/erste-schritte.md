# Erste Schritte

Diese Seite zeigt die drei Dinge, die Sie in Wikigit am häufigsten tun: eine
Seite **lesen**, eine **bearbeiten** und eine neue **erstellen**. Nichts davon
erfordert ein Konto.

## Lesen

Lesen ist einfach Stöbern. Jede Seite wird aus einem CDN geladen und in Ihrem
Browser dargestellt, daher laden Seiten schnell und sind stets aktuell. Achten
Sie beim Lesen auf das **Inhaltsverzeichnis**, auf **interne Links**, auf
**rote Links** (Seiten, die es noch nicht gibt) und auf den Reiter
**Diskussion**.

## Bearbeiten

1. Klicken Sie oben auf einer Seite auf **Bearbeiten** (oder auf den kleinen
   **edit**-Link neben einer Überschrift).
2. Tippen Sie in [[w:Markdown|Markdown]]. Eine **Live-Vorschau** erscheint neben
   dem Text.
3. Schreiben Sie eine kurze **Zusammenfassung** Ihrer Änderung.
4. Klicken Sie auf **Veröffentlichen**.

Ihr Entwurf wird auf Ihrem Gerät gespeichert, während Sie tippen. Was nach dem
Veröffentlichen geschieht, hängt von der Seite und vom erworbenen Vertrauen ab:
vertrauenswürdige Änderungen an offenen Seiten gehen sofort live, alle anderen
durchlaufen eine kurze Prüfung.

## Eine Seite erstellen

Zwei einfache Wege: Folgen Sie einem **roten Link** zu einer noch nicht
existierenden Seite, oder nutzen Sie den **Erstellungsassistenten**, der einen
Titel entgegennimmt, die Adresse vorschaut und eine Startvorlage anbietet. Die
neue Seite existiert sofort beim Speichern, ohne Neuerstellung.

## Wer Sie beim Bearbeiten sind

Sie melden sich nicht an, aber jede Änderung wird **signiert**: standardmäßig
**anonym** unter einem stabilen Pseudonym wie `anon-3f9a2c`, das aus Ihrer
Netzwerkadresse über einen Einweg-Hash abgeleitet wird (die rohe Adresse wird nie
gespeichert) — oder **unter Ihrer GitHub-Identität**, wenn Sie sich anmelden.
